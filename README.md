# Полдень — новостной сайт на Next.js (многостраничный)

## Структура страниц

Теперь у каждого раздела — свой собственный URL:

- `/` — Главное (все новости)
- `/news/mir` — Мир
- `/news/rossiya` — Россия
- `/news/biznes` — Бизнес
- `/news/tehnologii` — Технологии
- `/news/nauka` — Наука
- `/news/sport` — Спорт
- `/news/kultura` — Культура
- `/news/obschestvo` — Общество

Переход между разделами — это настоящая навигация по ссылкам (`<Link>`),
а не переключение состояния на одной странице. Каждую такую страницу
можно открыть напрямую по ссылке, ей можно поделиться, она попадёт
в поисковики отдельно от остальных.

## Как это устроено

```
nextjs-news/
  lib/news.js                 ← общая логика: источники, категории, парсинг RSS
  app/
    layout.js                 ← один раз забирает новости на сервере, оборачивает всё
    context/NewsContext.js     ← общее хранилище новостей на все страницы
    components/SiteChrome.js   ← шапка, категории-ссылки, бегущая строка, меню
    components/NewsFeed.js     ← лента + фильтр по источнику + поиск (общий для всех страниц)
    page.js                    ← страница "Главное" (/)
    news/[category]/page.js    ← динамическая страница раздела (/news/mir и т.д.)
    api/news/route.js          ← ручное обновление (кнопка "Обновить сейчас")
    globals.css
  package.json
  next.config.mjs
```

Новости забираются на сервере один раз при заходе на сайт (в `layout.js`)
и переиспользуются на всех страницах через общий контекст — RSS не
скачивается заново при переходе между разделами. Кнопка "Обновить
сейчас" дёргает `/api/news` и обновляет данные везде сразу.

Если в самом RSS-фиде нет картинки, сервер дополнительно заходит на
страницу первых ~60 самых свежих новостей и берёт оттуда `og:image`
(это то изображение, которое сайт-источник показывает при репосте в
соцсетях — оно есть почти всегда). Из-за этого первая загрузка страницы
может занимать на пару секунд больше, зато фотографий становится
заметно больше. Следующие 5 минут результат отдаётся из кэша, так что
это не повторяется на каждый клик.

## Что добавилось для схожести с крупными новостными сайтами

- **Несколько историй сверху** — 1 крупная + 2 средние, а не одна большая
- **"В тренде"** — если одну и ту же новость в одно время (окно 8 часов)
  пишут 2+ разных источника, она попадает в сайдбар "В тренде" на главной
  (сравнение заголовков по совпадающим значимым словам)
- **Курс валют** ($/€) в шапке — берётся с cbr-xml-daily.ru (ЦБ РФ)
- **Точное время публикации** — теперь видно при наведении на время
  ("X мин назад")
- **"Читайте также"** — компактный список ещё нескольких новостей из
  того же раздела между хедлайнами и общей сеткой
- **Тёмная тема** — кнопка ☾/☀︎ в шапке, сохраняется в браузере
- **Переключатель RU/EN** — кнопка в шапке переводит весь интерфейс
  (категории, кнопки, подписи) и сами заголовки/анонсы новостей через
  бесплатный переводчик MyMemory (`app/api/translate`). Переводятся
  только видимые на экране новости (не все 700+ сразу) — экономит лимит
  API и работает быстро. Готовые переводы кэшируются в браузере на время
  сессии, повторно не переводятся.

  ⚠️ У MyMemory (бесплатный, без ключа) небольшой дневной лимит символов.
  Если он исчерпан, новости просто останутся на русском вместо перевода —
  сайт не сломается, но перевод новостей может временно не работать.
  Интерфейс (кнопки, категории) при этом всё равно переключается всегда,
  так как хранится локально и не зависит от внешнего API.

## Как загрузить на GitHub (через веб-интерфейс)

1. Создай репозиторий на github.com (например `nextjs-news`).
2. **Add file → Upload files**, перетащи все файлы и папки из архива,
   сохраняя структуру (`lib`, `app`, `app/context`, `app/components`,
   `app/news/[category]`, `app/api/news` должны остаться папками).
3. Commit changes.

## Как развернуть на Vercel

1. vercel.com → **Add New → Project** → выбери репозиторий.
2. Framework Preset определится сам как **Next.js**.
3. **Deploy**.

Дальше как обычно: `git add .` → `git commit` → `git push` → Vercel
пересоберёт сайт автоматически.

## Локальная проверка (необязательно)

```
npm install
npm run dev
```

Открой http://localhost:3000 и http://localhost:3000/news/sport
